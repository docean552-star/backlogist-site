Use Cases

Not just a task tracker.
A command center.

From raw ideas to shipped features. Every agile ceremony, automated.

01
Killer Feature

AI Meeting Host

backlogist doesn't record meetings — it runs them. Opens the call, reads today's agenda from backlog, asks each participant about their tasks, logs answers in real-time, escalates blockers, closes with summary and auto-created action items. Daily standups: 3 questions per person, blockers escalated instantly.

$ backlogist meeting --standup --channel general
02
Core

Idea → Task Plan

Throw in a raw idea. backlogist searches 14 sources for context, breaks it into subtasks, assigns to agents by affinity, sets blockers and dependencies. One command — distributed plan for the whole team.

$ backlogist create title:"Need a blog" --template feature
03
Built-in

Time Tracking

Timer starts on take, stops on release or close. Know exactly how long each task takes. Use real data to plan sprints instead of guessing.

$ backlogist #352 take // timer starts
04
Agile

Sprint Management

Create sprints with scope and deadlines. Track progress with real burndown charts — not spreadsheet theater. Auto-archive done tasks older than 30 days.

$ backlogist sprint create --deadline 2026-04-01 --scope 280,317
05
Intelligence

Ask in Plain Language

No filters needed. Ask questions like a human: "What blocks SEO?", "Who's overloaded?", "What shipped this week?" Claude answers from live backlog state.

$ backlogist ask "what blocks the SEO pipeline?"
06
Visualization

Gantt & Dependency Graphs

See the critical path instantly. Mermaid dependency graphs, Gantt charts with owner swimlanes. Free alternative to what every task tracker charges for.

$ backlogist graph // mermaid blocked_by visualization
07
Integrations

Meeting Transcripts → Tasks

Fireflies.ai transcribes your client call. backlogist extracts action items and creates tasks automatically. Discussed "need a blog"? Task created, assigned, knowledge assembled. Telegram bot sends daily digests, human gates via inline buttons, real-time status updates. Notion two-way sync keeps the whole team on the same page.

$ backlogist sync --notion --telegram-digest
08
Automation

Recurring Tasks

Weekly review? Monthly client report? Set it once — backlogist creates tasks on schedule. Or on-close: "Sprint 5" done → "Sprint 6" auto-created with inherited fields.

$ backlogist create --recurrence weekly --template review
Core Engine

Everything under the hood.
Nothing unnecessary.

Nine commands, six hooks, one AI reviewer. Each piece eliminates a specific failure mode.

⚙

Lifecycle Management

Create, take, release, update, close — with enforced state transitions. BACKLOG → READY → IN_PROGRESS → IN_REVIEW → DONE. Invalid transitions rejected with clear guidance.

🔎

Knowledge Assembly

14 sources searched in parallel. Specs, legacy code, case studies, HANDOFF archives. Claude filters for relevance. Over-inclusive by design.

✅

Modular AI Review

Prompt modules activate per task type. Code → spec validation. Integration → data flow checks. Structured JSON verdicts: PASS, FAIL, NEEDS_WORK.

⚡

Three-Tier Performance

Hot-path <30ms (zero imports). Standard <3s. Heavy up to 120s with progress indicators.

📑

Immutable Audit Trail

Every field change: who, when, what, why. Append-only. Queryable via history.

🔌

Hook Enforcement

Six hooks in Claude Code's tool lifecycle. Plans injected every call. Reviews invalidated on edits. Closures gated on verdicts.

🎯

Taskwarrior-Style Filters

Natural syntax: #280 for ID, owner:A for field match, status:~DONE for negation, +urgent for tags. Reads like English.
